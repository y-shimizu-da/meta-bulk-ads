# Meta広告 一括入稿ツール 操作マニュアル

## 概要

Meta（Facebook/Instagram）広告の画像クリエイティブを一括で入稿できるツールです。
画像ファイルを複数アップロードし、広告文・見出し・リンク先URLを入力するだけで、キャンペーン・広告セット・クリエイティブ・広告がまとめて作成されます。

### 主な機能
- 画像ファイルを複数同時にアップロード（ドラッグ&ドロップ対応）
- 1画像 = 1広告として自動作成
- 広告名は画像ファイル名から自動設定
- キャンペーン「DA‐入稿用キャンペーン」を自動作成（既存があれば再利用）
- URLパラメータ（utm_creative等）を自動付与
- Advantage+エンハンスは全てOFF
- 全広告はPAUSED（一時停止）状態で作成

---

## 事前準備

### 1. アクセストークンの取得

1. [Graph API Explorer](https://developers.facebook.com/tools/explorer/) にアクセス
2. 右側パネルの **「Metaアプリ」** で自社アプリを選択（「Graph API Explorer」のままにしない）
3. **「ユーザーまたはページ」** で「ユーザートークン」を選択
4. **「アクセス許可」** セクションで以下の権限を追加：
   - `ads_management`
   - `pages_show_list`
   - `pages_read_engagement`
5. **「Generate Access Token」** をクリック
6. Facebookのログイン・権限確認画面が表示されたら許可する
7. 生成されたトークン（EAAで始まる文字列）をコピー

> **注意**: トークンは約1〜2時間で期限切れになります。期限が切れた場合は再度生成してください。

### 2. 広告アカウントIDの確認

1. [Meta広告マネージャ](https://adsmanager.facebook.com/) にアクセス
2. URLを確認：`https://adsmanager.facebook.com/adsmanager/manage/campaigns?act=1234567890`
3. `act=` の後の数字をメモ → **`1234567890`**（数字のみ入力すればOK）

### 3. FacebookページIDの確認

1. 広告に使用するFacebookページを開く
2. **「基本データ」**（About）をクリック
3. ページ下部の **「ページID」** をメモ（数字のみ）

## 操作手順

### Step 1: 接続設定

1. ツールを開く
2. 以下の3つを入力：

| 項目 | 入力内容 | 例 |
|------|---------|-----|
| アクセストークン | Graph API Explorerで取得したトークン | `EAAxxxxxxx...` |
| 広告アカウントID | 数字のみ | `1234567890` |
| FacebookページID | 数字のみ | `123456789012345` |

3. **「広告を入稿する」** をクリック

### Step 2: 広告データの入力

#### 画像ファイル
- **ドラッグ&ドロップ** または **クリックしてファイル選択** で画像をアップロード
- 複数画像を一度に選択可能
- 対応形式：JPG, PNG
- 推奨サイズ：1080×1080px以上
- **画像のファイル名がそのまま広告名になります**（拡張子は除く）
  - 例：`summer_sale_01.jpg` → 広告名は `summer_sale_01`

#### 広告文
- 広告の本文テキストを入力（全広告共通）

#### 見出し
- 広告の見出しを入力（全広告共通）

#### リンク先URL
- 広告クリック時の遷移先URLを入力
- 例：`https://example.com/lp`

#### URLパラメータ
- デフォルト：`utm_source=meta&utm_medium=paid`
- 自動付与されるパラメータ：
  ```
  ?utm_creative=[広告名]&utm_source=meta&utm_medium=paid
  ```
- 必要に応じて編集可能

### Step 3: 入稿の実行

1. 画面下部に「画像○枚 → 広告○件を入稿」と表示されていることを確認
2. **「入稿を開始」** をクリック
3. 進捗画面に切り替わり、以下の順序で処理が進みます：
   - キャンペーン作成（または既存キャンペーンの再利用）
   - 広告セット作成
   - 画像アップロード × N件
   - クリエイティブ作成 × N件
   - 広告作成 × N件

4. 全件完了後、結果が表示されます
5. **「新しい入稿を開始」** をクリックすると、入力画面に戻ります

---

## 自動作成される内容

| 項目 | 設定値 |
|------|--------|
| キャンペーン名 | DA‐入稿用キャンペーン |
| キャンペーン目的 | トラフィック（OUTCOME_TRAFFIC） |
| 広告セット名 | 広告セット_YYYYMMDD_HHMM |
| 日予算 | ¥2,000（広告セットレベル） |
| 最適化目標 | リンクのクリック |
| 入札戦略 | 最小コスト（上限なし） |
| ターゲティング | 日本、18〜65歳 |
| 開始日 | 翌日 0:00 |
| CTA | 詳しくはこちら（LEARN_MORE） |
| ステータス | 一時停止（PAUSED） |
| Advantage+エンハンス | 全てOFF |

---

## 入稿後の確認

1. [Meta広告マネージャ](https://adsmanager.facebook.com/) にアクセス
2. 「DA‐入稿用キャンペーン」を検索
3. キャンペーン内の広告セット・広告を確認
4. 必要に応じてターゲティングや予算を調整
5. 配信を開始する場合は、広告のステータスを **「アクティブ」** に変更

> **重要**: 全ての広告は「一時停止」状態で作成されます。配信を開始するには手動でアクティブにする必要があります。

## トラブルシューティング

### 「Application request limit reached」エラー
- **原因**: Meta APIのレート制限に達した
- **対策**: 5〜10分待ってから再度実行してください

### 「Object does not exist」エラー
- **原因**: 広告アカウントIDの入力ミス（スペース混入など）
- **対策**: `act_` の後にスペースが入っていないか確認してください

### 「クリエイティブ投稿は開発モードのアプリにより作成されたものです」
- **原因**: Facebookアプリが開発モード
- **対策**: [Meta開発者ダッシュボード](https://developers.facebook.com/) でアプリを「公開」モードに切り替えてください

### アクセストークンの期限切れ
- **症状**: 「Invalid OAuth access token」等のエラー
- **対策**: Graph API Explorerで新しいトークンを生成してください

### 広告が管理画面に表示されない
- 広告マネージャの検索フィルタを確認してください
- 「DA‐入稿用キャンペーン」で検索、または「一時停止」フィルタを含めて表示してください

---

## 注意事項

- トークンは**絶対に外部に共有しないでください**（Facebook アカウントへのアクセス権を持ちます）
- 入稿後は必ず広告マネージャで内容を確認してから配信を開始してください
- 大量入稿時（10件以上）はレート制限に注意してください（ツール側で自動的に2秒間隔を空けています）
